# Humanize Writing

## The core diagnosis

AI writing has six tells. Check for all of them before editing:

1. **Zero first-person** — no "I think", "in my view", "honestly", "to me"
2. **Structural colons** — colons used as "behold what follows" punctuation mid-sentence
3. **Checklist rebuttals** — "On X:" / "First:" / "Regarding X:" patterns that read like a list in disguise
4. **Textbook phrasing** — formal words a real person wouldn't say out loud
5. **Perfect qualification** — every claim hedged equally, no real conviction
6. **Overly smooth transitions** — paragraphs glide together too perfectly

## Workflow

1. Read the full draft and flag every instance of the six tells above
2. Apply fixes using the patterns in `references/patterns.md`
3. Read the result aloud — if it sounds like someone talking, it's done; if it still sounds like a report, repeat

## Key rules

**First-person goes at high-stakes moments only.** Not every sentence. Target:
- After stark numbers or surprising facts ("Honestly, that is a remarkable outcome")
- When landing the main argument ("In my view, that is not a side effect. It is the design.")
- In the rebuttal ("I think the legality argument is the weakest of the three")
- In the conclusion ("To me, a program that works by discouraging people is not a fair process")

**Short sentences land punches.** After a long sentence building an argument, one short sentence delivers the verdict. "It is the design." "It is a liability shield." "Not much of a right." Never stack more than two short sentences in a row.

**Numbers do the drama — prose stays plain.** Lead with the number, follow with the plain-English meaning. Don't bury figures in analytical prose.

**Transitions should feel earned, not engineered.** Replace "The X problem follows directly from this" with "The X problem is just as serious" or just start the new paragraph with the point.
